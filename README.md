# BM25 Gets Its Revenge: companion experiment

Small retrieval experiment for the article
[*Before Another RAG Hop, Try Compiling the Query for BM25*](https://medium.com/@cagatay.akcamm/before-another-rag-hop-try-compiling-the-query-for-bm25-9652b5f4cee6).
Rebuilds **one slice** of Meta's [SIRA](https://arxiv.org/abs/2605.06647) pipeline:
query-side vocabulary prediction, document-frequency filtering, and weighted BM25.

**Not included:** offline document enrichment (SIRA stage 1) or LLM reranking.

## Quick start

```bash
pip install -r requirements.txt
python experiment.py          # metrics → results.json
python compare_candidates.py  # hand vs Sonnet → results_llm_comparison.json
python figures.py             # seven PNGs → figures/
python build_notebook.py      # optional: rebuild bm25_revenge.ipynb
```

Or open `bm25_revenge.ipynb` in Jupyter.

First run downloads `all-MiniLM-L6-v2` (~90 MB). After that, evaluation finishes
in under a minute on a laptop. No API key required.

## Methodology notes

- **Candidate terms are hand-authored**, not model-generated. They simulate what a
  well-prompted LLM might propose and were frozen before scoring. Corpus-aware
  expansion scoring 1.000 on all metrics reflects that choice, not a guarantee
  about real LLM output.
- An optional notebook cell regenerates candidates via the Anthropic API
  (`claude-sonnet-4-6`). The frozen file `llm_candidates.json` in this repo
  was generated once with that model; `python compare_candidates.py` contrasts
  it against the hand-authored lists.
- Dense (MiniLM) beats vanilla BM25 overall on this corpus (MRR 0.917 vs 0.839).
  The experiment isolates where DF-filtered expansion fixes BM25's worst failures.
